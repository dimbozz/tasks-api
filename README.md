Реализуйте REST API для управления списком задач (to‑do). Сервис работает в памяти (без БД) и отдаёт/принимает только JSON.

Требуемые эндпоинты
· GET /tasks — получить список задач.
· POST /tasks — создать задачу.
· GET /tasks/{id} — получить задачу по идентификатору.
· PUT /tasks/{id} — обновить задачу целиком.
· DELETE /tasks/{id} — удалить задачу.

Контракт данных

```
Task {
  "id": number,        // генерируется на сервере
  "title": string,     // обязательное поле
  "done": boolean,     // статус выполнения
  "created_at": string // ISO8601, по желанию
}
```

Нефункциональные требования

· Ответы и ошибки — в JSON. Заголовок: Content-Type: application/json.
· Коды статусов: 200/201/204 для успеха; 400 для неверных данных; 404 для отсутствующих ресурсов; 405 при неподдерживаемом методе; 500 при внутренних ошибках.
· Единый формат ошибок, например: { “error”: “сообщение” }.
· Хранение: in‑memory (синхронизация по необходимости).
· Логи входящих запросов (минимум: метод и путь).
· PRO по желанию: GET /health (health‑check) и описание API (таблица эндпоинтов в README или OpenAPI‑файл).
