Инструкция к выполнению
1. Склонируйте/создайте проект из скелета выше и инициализируйте go mod init tasks-api.
2. Реализуйте модель Task и in‑memory хранилище по интерфейсу Storage (самостоятельная реализация, потокобезопасность обязательна).
3. Настройте HTTP‑сервер (net/http или выбранный роутер). Зарегистрируйте маршруты коллекции и элемента.
4. Реализуйте хендлеры для всех методов. Обеспечьте корректные коды статусов и заголовки.
5. Сведите все ответы к JSON (включая ошибки). Введите единый формат ошибок.
6. Добавьте базовое логирование запросов.
7. Проведите ручное тестирование: выполните для каждого эндпоинта по одному корректному и по одному ошибочному сценарию, зафиксируйте команды curl и ответы.
8. Подготовьте README: как запустить, таблицу эндпоинтов, примеры запрос/ответ.
9. (Опционально) Добавьте GET /health и/или описание API (таблица, Postman‑коллекция или OpenAPI‑файл). Добавьте синхронизацию при доступе из нескольких обработчиков.
